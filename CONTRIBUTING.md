# Contributing

Thanks for your interest in contributing!

See **[docs/CONTRIBUTING.md](docs/CONTRIBUTING.md)** for the full contribution governance — rule proposal flow, skill conventions, semver bump rules, and PR template.

## Quick-start

1. **Issue first**: open an issue (use Rule Proposal or Skill Proposal template) describing what you want to add and why
2. **Triage**: maintainer applies the 4-question checklist (Generalize / Stable / No project paths / Battle-tested)
3. **PR**: implement, follow `docs/CONTRIBUTING.md`, fill PR template, bump VERSION + CHANGELOG per semver
4. **Review**: maintainer reviews against `rules/rule-change-process.md` §5 review matrix
